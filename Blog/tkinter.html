üß± Core Setup & Main Window

The essential steps for creating the main application window and starting the event loop.

The Core Loop: import tkinter as tk, tk.Tk(), root.mainloop()

Description: Every Tkinter app needs: 1) the **import**; 2) the **root window** (`tk.Tk()`); 3) the **event loop** (`mainloop()`), which keeps the window open and responsive to user interaction.

import tkinter as tk

# 1. Create the main window
root = tk.Tk()
root.title("My First Tkinter App")
root.geometry("400x300") # Width x Height

# 2. Start the event loop (must be the last line!)
root.mainloop()

Secondary Window: tk.Toplevel(master)

Description: Creates a new, independent top-level window (a pop-up or dialog box) separate from the main root window.

Event Binding: widget.bind(event, handler)

Description: Attaches a function (`handler`) to a specific event (like a mouse click or key press) on a widget or the root window.

def on_click(event):
    print("Window clicked!")

root.bind('', on_click) # Binds left-mouse click

üìã Basic Input & Display Widgets

The fundamental building blocks for showing text, receiving input, and triggering actions.

Display Text: tk.Label

Class: tk.Label(master, text="...")

Purpose: Used to display static text or images. Often used to label other input fields.

Action Trigger: tk.Button

Class: tk.Button(master, text="...", command=function)

Purpose: Creates a clickable button. The `command` option specifies the Python function to run when the button is pressed.

def handle_click():
    print("Button pressed!")

btn = tk.Button(root, text="Click Me", command=handle_click)
btn.pack()

Single-Line Input: tk.Entry

Class: tk.Entry(master, width=...)

Purpose: Provides a text field for the user to type in a single line of text (e.g., username, password, number).

Method: Use entry.get() to retrieve the text entered by the user.

Multi-Line Input: tk.Text

Class: tk.Text(master, height=...)

Purpose: Used for large, multi-line text areas, like a message body or code editor. Offers advanced text formatting capabilities.

üìê Layout Managers (The 3 Ways to Organize)

You must use one of these methods to place a widget inside its master window or frame. They cannot be mixed within the same master window.

1. Pack Manager: widget.pack()

Concept: Simplest layout. Widgets are organized in blocks before they are placed in the parent widget. Options like `side='left'`, `fill='x'`, or `expand=True` control placement.

btn1.pack(side='left') # Place button on the left
btn2.pack(fill='x')    # Stretch button horizontally

2. Grid Manager: widget.grid(row=R, column=C)

Concept: The most common layout. Organizes widgets in a table-like structure of rows and columns. Uses `rowspan` and `columnspan` for merging cells.

label.grid(row=0, column=0)
entry.grid(row=0, column=1, padx=5, pady=5) # Add padding

3. Place Manager: widget.place(x=X, y=Y)

Concept: Absolute positioning. Places a widget at specific coordinates (pixels) relative to the top-left corner of the parent. **Generally discouraged** as it doesn't handle resizing well.

‚ÜîÔ∏è Variable-Bound Widgets (State Control)

Widgets that rely on a special Tkinter Variable class to store their state (value, text, etc.) outside the widget itself.

Tkinter Variables: tk.StringVar(), tk.IntVar(), tk.DoubleVar(), tk.BooleanVar()

Description: These objects act as containers for data. Methods include .get() to read the value and .set(value) to change the value programmatically.

Boolean Input: tk.Checkbutton

Class: tk.Checkbutton(..., variable=tk.IntVar())

Purpose: Creates a checkbox. Its state (checked/unchecked) is bound to a `tk.IntVar` (1 for checked, 0 for unchecked).

Exclusive Choice: tk.Radiobutton

Class: tk.Radiobutton(..., variable=V, value=N)

Purpose: Creates a set of buttons where only one can be selected at a time. All buttons in the group must share the same `variable`.

Slider: tk.Scale

Class: tk.Scale(..., from_=0, to=100)

Purpose: Creates a slider widget (horizontal or vertical) for selecting numeric values within a defined range.

‚öôÔ∏è Advanced Widgets

Widgets used for structuring the layout and handling complex data display.

Container: tk.Frame

Class: tk.Frame(master)

Purpose: An invisible rectangular container. Used to group related widgets together and apply a separate layout manager (Grid, Pack) to that group, improving organization.

Drawing Surface: tk.Canvas

Class: tk.Canvas(master)

Purpose: Provides a drawing surface for complex graphics, lines, arcs, images, and other custom shapes. Often used in conjunction with Turtle graphics.

Methods: canvas.create_line(), canvas.create_oval(), canvas.create_text().

Top Menu Bar: tk.Menu

Purpose: Used to create the standard top-level menu bar (File, Edit, Help, etc.) and cascading sub-menus.

# Create the main menu bar
menu_bar = tk.Menu(root)
root.config(menu=menu_bar)

# Create a sub-menu
file_menu = tk.Menu(menu_bar, tearoff=0)
file_menu.add_command(label="Exit", command=root.quit)

# Add the sub-menu to the bar
menu_bar.add_cascade(label="File", menu=file_menu)
